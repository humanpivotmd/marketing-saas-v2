# v3 향후 계획

## 1. 미구현 기능

### 발행 예약 자동화
- 현재: 발행 예정일(scheduled_date)을 수동 설정하고 캘린더에 표시만 함
- 목표: 예정일에 자동으로 SNS 채널에 발행 (schedules 테이블 + cron job)
- 필요: 각 플랫폼 API 연동 (네이버 블로그 API, Instagram Graph API, Threads API, Facebook Pages API)

### 성과 추적
- 현재: content_metrics 테이블 존재하나 데이터 수집 미구현
- 목표: 발행된 콘텐츠의 조회수/좋아요/댓글/공유 수집, 대시보드 표시
- 필요: 각 플랫폼 Insights API 연동 + 주기적 수집 cron

### 소셜 로그인
- 현재: 이메일/비밀번호 로그인만 지원
- 목표: Google, Kakao, Naver 소셜 로그인
- 필요: Supabase Auth Provider 설정 또는 NextAuth 도입

### 팀 협업
- 현재: plans 테이블에 team_member_limit 필드 존재하나 팀 기능 미구현
- 목표: 팀 초대, 역할 기반 접근 제어, 공유 프로젝트

## 2. 개선 예정

### Redis Rate Limit
- 현재: 인메모리 Map 기반 (서버 재시작 시 초기화, 멀티 인스턴스 미지원)
- 목표: Redis 기반 분산 rate limit
- 우선순위: 높음 (멀티 인스턴스 배포 시 필수)

### system role 분리
- 현재: `SplitPrompt` 인터페이스(system/user)가 정의되어 있으나 실제 API 호출에서는 단일 user 메시지로 전송
- 목표: Claude API 호출 시 system 파라미터 활용으로 프롬프트 품질 향상
- 효과: 시스템 지시와 사용자 입력 분리로 인젝션 저항력 강화

### 유저 API 키 등록
- 현재: 서버 환경변수의 ANTHROPIC_API_KEY 단일 키 사용
- 목표: 유저가 자신의 API 키를 등록하여 사용 (비용 분담)
- 필요: users 테이블에 encrypted API key 컬럼, ENCRYPTION_KEY로 암호화/복호화

### 프롬프트 A/B 테스트
- 현재: admin_prompts 테이블에 version/traffic_ratio 필드 존재하나 실제 A/B 분기 미구현
- 목표: 프롬프트 버전별 성과 비교, 자동 최적화

### 콘텐츠 편집기
- 현재: 생성된 콘텐츠를 textarea 수정 요청으로만 재생성
- 목표: 인라인 편집 + SEO 점수 실시간 반영 + 마크다운 에디터

### industries 중분류/소분류
- 현재: 14개 대분류만 seed, 중분류는 관리자가 수동 추가
- 목표: 주요 업종별 중분류 사전 seed 데이터 추가

## 3. 검증팀 지적 미해결 항목

### 보안
- 인메모리 rate limit는 프로덕션 부적합 -> Redis 전환 필요
- sanitizePromptInput 패턴 목록이 제한적 -> 더 광범위한 인젝션 패턴 추가 필요
- system/user 프롬프트 미분리 -> 프롬프트 인젝션 표면 확대 가능

### 안정성
- SSE 스트림 중단 시 부분 생성 결과 처리 (현재: 채널 완료 시마다 content_ids 업데이트로 부분 대응)
- 초안 중복 생성 방지 로직은 있으나, pipeline(STEP5) 중복 호출 방지는 미흡
- video_scripts 중복 삭제 시 최신 건 이외 전부 삭제하는 방식 -> 히스토리 유실

### UX
- STEP4 초안이 사용자에게 노출되지 않아 초안 수정 불가
- 이미지 프롬프트 결과를 실제 이미지로 생성하는 기능 미연동
- 영상 스크립트 결과를 영상으로 전환하는 기능 미제공
- 모바일 Stepper 가로 스크롤 시 현재 단계가 뷰포트 밖일 수 있음
