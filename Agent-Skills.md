# Agent Skills - 워크플로우 실행 가이드

## 개요
Agent Skills는 AI 에이전트의 작업 실행을 표준화된 단계로 나누어 체계적인 개발을 지원합니다.

## 워크플로우 개요

```
DEFINE → PLAN → BUILD → VERIFY → REVIEW → SHIP
   ↓      ↓      ↓       ↓       ↓       ↓
 /spec  /plan  /build  /test  /review /ship
```

## 각 단계 상세

### 1. /spec - 요구사항 정의
**목적**: 작업의 범위와 요구사항을 명확히 정의

**역할**:
- 사용자 요청 분석
- 기능 요구사항 추출
- 제약 조건 식별
- 위험 요소 평가

**입력**:
- 사용자 요청 텍스트
- 현재 시스템 상태
- 관련 문서/코드

**출력**:
- 요구사항 명세서 (Markdown)
- **상세 영향 분석 리포트**:
  - 파일 의존성 맵 (import/export 관계)
  - UI/UX 영향도 (레이아웃, 플로우, 접근성)
  - 프로세스 영향도 (비즈니스 로직, 워크플로우)
  - 함수 영향도 (호출 체인, 인터페이스 변경)
  - DB 영향도 (스키마, 마이그레이션, 데이터 호환성)
  - API 영향도 (엔드포인트, 요청/응답 포맷)
  - 외부 의존성 영향도 (라이브러리, 서드파티 서비스)
- 위험도 평가 (🔴🟡🟢)
- 예상 작업 시간

**체크리스트**:
- [ ] 기능 요구사항 완전성
- [ ] 비기능 요구사항 포함 (성능, 보안, 사용성)
- [ ] **복합 영향 분석 완료**:
  - [ ] UI 컴포넌트 영향 분석 (화면, 컴포넌트, 상태)
  - [ ] 비즈니스 로직 영향 분석 (프로세스, 규칙, 계산)
  - [ ] 기술적 영향 분석 (함수, 클래스, 모듈)
  - [ ] 데이터 영향 분석 (DB, 캐시, 파일)
  - [ ] 통합 영향 분석 (API, 외부 서비스)
  - [ ] 테스트 영향 분석 (단위, 통합, E2E)
- [ ] 제약 조건 명시
- [ ] 의존성 식별
- [ ] 테스트 가능성 확인

**실행 규칙**:
- CLAUDE.md의 금지 영역 확인
- 기존 코드 grep으로 영향 범위 파악
- API/DB 스키마 변경 필요성 평가

### 2. /plan - 작업 계획 수립
**목적**: 실행 가능한 작업 단위로 분해

**역할**:
- 작업 브레이크다운
- 우선순위 설정
- 리소스 할당
- 의존성 관리

**입력**:
- /spec 결과물
- 팀 리소스 상태
- 프로젝트 일정

**출력**:
- 작업 단위 리스트 (최대 2시간)
- 우선순위 매트릭스
- 의존성 그래프
- 타임라인
- 롤백 계획

**체크리스트**:
- [ ] 작업 단위 적정 크기
- [ ] 병렬 실행 가능성
- [ ] 리스크 완화 전략
- [ ] 테스트 계획 포함
- [ ] 문서화 계획

**실행 규칙**:
- 3개 파일 초과 수정 금지
- 의존성 순서 준수
- 코드 리뷰 포인트 사전 정의

### 3. /build - 코드 구현
**목적**: 요구사항을 코드로 구현

**역할**:
- 코드 작성
- 타입 정의
- 단위 테스트 작성
- 문서화

**입력**:
- 승인된 작업 계획
- 기존 코드베이스
- 디자인 가이드라인

**출력**:
- 구현된 코드
- TypeScript 타입
- 단위 테스트
- API 문서

**체크리스트**:
- [ ] 컴파일 에러 없음
- [ ] 린팅 규칙 준수
- [ ] 코드 커버리지 80%+
- [ ] JSDoc 주석 완성
- [ ] 임시 코드 제거

**실행 규칙**:
- CLAUDE.md 품질 기준 준수
- 기존 패턴/규칙 따르기
- 변경 파일 3개 제한
- 커밋 전 로컬 테스트

### 4. /test - 검증 및 테스트
**목적**: 구현의 정확성과 품질 검증

**역할**:
- 단위 테스트 실행
- 통합 테스트 수행
- 성능 테스트
- 보안 스캔

**입력**:
- 구현된 코드
- 테스트 케이스
- 성능 기준

**출력**:
- 테스트 결과 리포트
- 커버리지 리포트
- 성능 측정 결과
- 보안 스캔 리포트

**체크리스트**:
- [ ] 빌드 성공
- [ ] 단위 테스트 통과
- [ ] 통합 테스트 통과
- [ ] E2E 테스트 통과
- [ ] 성능 기준 만족
- [ ] 보안 취약점 없음

**실행 규칙**:
- CLAUDE.md의 Ralph Loop 준수
- 실패 시 즉시 중단
- 커버리지 임계값 확인
- 크로스 브라우저 테스트

### 5. /review - 코드 리뷰
**목적**: 코드 품질과 표준 준수 검토

**역할**:
- 코드 품질 평가
- 보안 취약점 검사
- 성능 최적화 검토
- 표준 준수 확인

**입력**:
- 구현 코드 + 테스트 결과
- 코드 리뷰 체크리스트
- 프로젝트 표준

**출력**:
- 리뷰 코멘트
- 개선 제안
- 승인/거부 결정
- 리팩토링 계획

**체크리스트**:
- [ ] 코드 중복 제거
- [ ] 네이밍 일관성
- [ ] 보안 취약점 없음
- [ ] 성능 최적화 적용
- [ ] 접근성 준수
- [ ] 문서화 완성

**실행 규칙**:
- CLAUDE.md 품질 기준 적용
- 자동화된 리뷰 도구 활용
- 팀 표준 준수 확인
- 개선 제안 구체화

### 6. /ship - 배포 및 모니터링
**목적**: 프로덕션 환경으로 안전한 배포

**역할**:
- 배포 실행
- 모니터링 설정
- 롤백 준비
- 사용자 피드백 수집

**입력**:
- 리뷰 승인된 코드
- 배포 환경 설정
- 모니터링 도구

**출력**:
- 배포 로그
- 모니터링 대시보드
- 롤백 스크립트
- 배포 리포트

**체크리스트**:
- [ ] 브랜치 전략 준수
- [ ] 커밋 규칙 준수
- [ ] CI/CD 통과
- [ ] DB 마이그레이션 완료
- [ ] 환경 설정 확인
- [ ] 모니터링 활성화

**실행 규칙**:
- CLAUDE.md 배포 정책 준수
- 점진적 롤아웃
- 모니터링 임계값 설정
- 긴급 롤백 준비

## 단계별 전환 규칙

### /spec → /plan
- 요구사항이 완전하게 정의됨
- 위험도가 평가됨
- 작업 범위가 명확함

### /plan → /build
- 작업 단위가 승인됨
- 의존성이 해결됨
- 리소스가 할당됨

### /build → /test
- 코드가 컴파일됨
- 기본 기능이 작동함
- 단위 테스트가 작성됨

### /test → /review
- 모든 테스트가 통과됨
- 커버리지가 만족됨
- 성능 기준이 충족됨

### /review → /ship
- 코드 리뷰가 승인됨
- 개선사항이 적용됨
- 배포 준비가 완료됨
