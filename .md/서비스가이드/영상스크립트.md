# STEP7: 영상 스크립트 가이드

> 블로그/SNS 콘텐츠를 기반으로 숏폼(세로 9:16) 또는 일반(가로 16:9) 영상의 스토리보드와 나레이션 스크립트를 생성한다.

---

## 1. 개요

| 항목 | 내용 |
|------|------|
| 파이프라인 위치 | STEP7 (최종 단계) |
| API | `/api/generate/video-script` |
| AI 모델 | `claude-sonnet-4-20250514` / max_tokens: 2048 |
| 출력 | 스토리보드(씬별 화면/나레이션/자막) + BGM 제안 + 후킹 멘트 |
| DB 저장 | `video_scripts` 테이블 |

---

## 2. API 상세

### `/api/generate/video-script`

```
POST /api/generate/video-script
Content-Type: application/json
Authorization: Bearer {token}

{
  "project_id": "uuid-...",
  "format": "short",
  "target_channel": "youtube",
  "scene_count": 4,
  "scene_duration": 5,
  "image_style": "photo",
  "style_detail": "모던"
}
```

**요청 파라미터:**

| 필드 | 필수 | 기본값 | 타입 | 설명 |
|------|------|--------|------|------|
| project_id | O | - | uuid | 프로젝트 ID |
| format | - | short | `short` \| `normal` | 숏폼/일반 |
| target_channel | - | youtube | string | 대상 플랫폼 |
| scene_count | - | 4 | number | 장면 수 |
| scene_duration | - | 5 | number | 장면당 시간(초) |
| image_style | - | photo | string | 이미지 스타일 |
| style_detail | - | 모던 | string | 세부 스타일 |

**총 영상 길이** = scene_count x scene_duration (초)

---

## 3. 영상 포맷

### 숏폼 (short)

| 항목 | 규격 |
|------|------|
| 화면 비율 | 세로 9:16 |
| 총 길이 | 15~60초 권장 |
| 장면 수 | 3~6개 |
| 장면당 시간 | 3~10초 |
| 적합 채널 | YouTube Shorts, Instagram Reels, TikTok |

### 일반 (normal)

| 항목 | 규격 |
|------|------|
| 화면 비율 | 가로 16:9 |
| 총 길이 | 60초~5분 권장 |
| 장면 수 | 5~12개 |
| 장면당 시간 | 5~15초 |
| 적합 채널 | YouTube, 블로그 임베드, 웹사이트 |

## 5. B2B vs B2C 영상 전략

### B2B 영상 원칙

| 요소 | 가이드 |
|------|--------|
| 톤 | 설명형/데모 중심, 전문적 내레이션 |
| 비주얼 | 데이터 시각화(그래프, 차트), 대시보드 화면 |
| 장면 구성 | 제품/서비스 데모, 사례 발표, 전문가 인터뷰 형식 |
| CTA | "무료 상담", "데모 신청", "백서 다운로드" |

**B2B 스토리보드 구조 (숏폼):**

| 씬 | 내용 | 목적 |
|----|------|------|
| 1 | 문제 제기 + 데이터 | 후킹 |
| 2 | 솔루션 소개 | 제품/서비스 설명 |
| 3 | 성과 데이터/사례 | 신뢰 구축 |
| 4 | CTA | 전환 유도 |

### B2C 영상 원칙

| 요소 | 가이드 |
|------|--------|
| 톤 | 감성적이고 빠른 장면 전환 |
| 후킹 | 첫 3초 강력한 후킹 (질문/충격/공감) |
| 비주얼 | 인물 중심 라이프스타일 장면 |
| 효과 | 트렌디한 BGM, 자막 효과 |
| CTA | "지금 바로", "링크 클릭", "댓글로 알려주세요" |

**B2C 스토리보드 구조 (숏폼):**

| 씬 | 내용 | 목적 |
|----|------|------|
| 1 | 후킹 (질문/충격적 사실) | 이탈 방지 |
| 2 | 공감 장면 + 문제 | 몰입 |
| 3 | 해결책/팁 | 가치 전달 |
| 4 | CTA + 감성 마무리 | 행동 유도 |

---

## 6. 씬 구성 요소

각 씬에 포함되는 요소:

| 필드 | 설명 | 예시 |
|------|------|------|
| scene | 씬 번호 | 1, 2, 3... |
| duration | 장면 길이(초) | 5 |
| visual | 화면 설명 (촬영/편집 가이드) | "밝은 카페 전경, 슬로우 줌인" |
| narration | 나레이션 텍스트 | "카페 인테리어, 이 3가지가 핵심입니다" |
| text_overlay | 화면 자막 | "카페 인테리어 3가지 팁" |
| transition | 장면 전환 효과 | fade, slide, cut, zoom |

추가 메타 정보:

| 필드 | 설명 |
|------|------|
| bgm_suggestion | 배경음악 분위기 추천 |
| hook | 첫 3초 후킹 멘트 |

## 7. 타겟 채널별 가이드

| 채널 | 포맷 | 길이 | 특징 |
|------|------|------|------|
| YouTube Shorts | short (9:16) | 15~60초 | 후킹 3초 필수, 자막 중심 |
| YouTube | normal (16:9) | 1~5분 | 설명형, 챕터 구분 |
| Instagram Reels | short (9:16) | 15~90초 | 트렌디, 빠른 전환 |
| TikTok | short (9:16) | 15~60초 | 트렌드 음악, 유머 |

## 8. 업종별 영상 전략

### 음식업

| 포맷 | 추천 구성 |

| 숏폼 | 조리 과정 타임랩스, 먹방 클로즈업, ASMR |

### 의료 (의료법 주의)

| 포맷 | 추천 구성 |
|------|-----------|
| 숏폼 | 건강 팁 3가지, Q&A 형식 |
| 일반 | 진료 과정 설명, 의료진 소개, 시설 투어 |

### 뷰티/헬스

| 포맷 | 추천 구성 |
|------|-----------|
| 숏폼 | 시술 전후 비교, 루틴 소개, 팁 |
| 일반 | 시술 과정 + 인터뷰 + 결과 |

### IT/SaaS

| 포맷 | 추천 구성 |
|------|-----------|
| 숏폼 | 기능 하이라이트, 데이터 시각화 |
| 일반 | 제품 데모, 사용법 튜토리얼, 고객 인터뷰 |

## 9. DB 스키마 (video_scripts)

| 필드 | 타입 | 설명 |
|------|------|------|
| id | uuid | PK |
| project_id | uuid | 프로젝트 FK |
| user_id | uuid | 사용자 FK |
| format | text | short / normal |
| target_channel | text | youtube, instagram 등 |
| scene_count | int | 장면 수 |
| scene_duration | int | 장면당 시간(초) |
| total_duration | int | 총 길이(초) |
| image_style | text | photo / illustration 등 |
| style_detail | text | 세부 스타일 |
| storyboard | jsonb | 씬별 스토리보드 배열 |
| full_script | text | 전체 스크립트 원문 |
| status | text | generated / used |

**중복 방지:** 새 스크립트 생성 시 같은 프로젝트의 이전 스크립트는 자동 삭제.

---

## 10. 프로젝트 step_status 업데이트

영상 스크립트 생성 완료 시:
- `step_status.s7` → `'completed'`
- 이것으로 7단계 파이프라인 전체가 완료된다.
