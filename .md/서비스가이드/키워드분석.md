# STEP1: 키워드 분석 가이드

> 파이프라인 첫 단계. 네이버 광고 API + DataLab 트렌드를 결합하여 키워드의 경쟁도/기회를 등급(A+~D-)으로 산출한다.

---

## 1. 개요

사용자가 입력한 키워드(최대 10개)에 대해 다음 4가지 분석을 수행한다.

| 분석 항목 | API 엔드포인트 | 메서드 | 설명 |
|-----------|---------------|--------|------|
| 검색량 조회 | `/api/keywords/naver` | POST | 네이버 광고 API 월간 검색량 |
| 등급 분석 | `/api/keywords/grade` | POST | A+~D- 등급 + 추천도 산출 |
| 트렌드 | `/api/keywords/trends` | POST | DataLab 기반 월별 검색 추이 |
| 기회 키워드 | `/api/keywords/opportunities` | GET | 업종별 기회 키워드 추천 |
| SEO 점수 | `/api/keywords/seo-score` | POST | 작성된 콘텐츠의 SEO 점수 |

---

## 2. 네이버 키워드 검색량 조회

### API: `/api/keywords/naver`

```
POST /api/keywords/naver
Content-Type: application/json
Authorization: Bearer {token}

{
  "keywords": ["마케팅 자동화", "콘텐츠 마케팅"]  // 1~5개
}
```

**요청 파라미터:**

| 필드 | 타입 | 제약 | 설명 |
|------|------|------|------|
| keywords | string[] | min 1, max 5, 각 최대 200자 | 조회할 키워드 목록 |

**응답 데이터:**

| 필드 | 설명 |
|------|------|
| monthlyPc | PC 월간 검색량 |
| monthlyMobile | 모바일 월간 검색량 |
| pcClickCnt | PC 클릭수 |
| mobileClickCnt | 모바일 클릭수 |

**활용:** 검색량 raw 데이터를 등급 분석의 입력값으로 사용한다.

## 3. 키워드 등급 시스템 (A+~D-)

### API: `/api/keywords/grade`

```
POST /api/keywords/grade
Content-Type: application/json
Authorization: Bearer {token}

{
  "keywords": ["키워드1", "키워드2", ...]  // 1~10개
}
```

**사용량 제한:** `keyword_analyze` 타입으로 체크. 플랜별 월간 분석 횟수 제한 적용.

### 등급 산출 공식

4가지 지표를 가중 합산하여 총점(0~100)을 산출하고, 총점 구간에 따라 등급을 부여한다.

#### 가중치

| 지표 | 가중치 | 설명 |
|------|--------|------|
| 검색량 점수 (search) | 35% | 월간 검색량 구간별 점수 |
| 포화도 점수 (saturation) | 35% | 월간 발행량 / 월간 검색량 비율 |
| 노출 지속일 (exposure) | 20% | 콘텐츠 상위 노출 유지 기간 |
| CPC (cpc) | 10% | 평균 클릭 비용 |

#### 검색량 점수 기준

| 월간 검색량 | 점수 | 의미 |
|-------------|------|------|
| 100,000 이상 | 20 | 대형 키워드, 경쟁 극심 |
| 50,000~99,999 | 30 | 대형 키워드 |
| 10,000~49,999 | 50 | 중대형 키워드 |
| 5,000~9,999 | 70 | 중형 키워드 |
| 1,000~4,999 | 85 | 중소형, 추천 |
| 500~999 | 90 | 소형, 진입 적합 |
| 100~499 | 80 | 니치 키워드 |
| 50~99 | 60 | 검색량 부족 |
| 10~49 | 40 | 매우 적음 |
| 10 미만 | 15 | 거의 없음 |

#### 포화도 점수 기준

포화도 = (월간 발행량 / 월간 검색량) x 100

| 포화도(%) | 점수 | 의미 |
|-----------|------|------|
| 5 이하 | 95 | 블루오션 |
| 6~10 | 85 | 경쟁 매우 낮음 |
| 11~30 | 75 | 경쟁 낮음 |
| 31~50 | 65 | 경쟁 보통 |
| 51~100 | 55 | 경쟁 다소 높음 |
| 101~300 | 40 | 경쟁 높음 |
| 301~1,000 | 25 | 레드오션 |
| 1,001~5,000 | 15 | 극심 |
| 5,000 초과 | 5 | 포화 |

#### 노출 지속일 점수

| 기간 | 점수 |
|------|------|
| 90일 이상 | 100 |
| 60~89일 | 85 |
| 30~59일 | 70 |
| 14~29일 | 55 |
| 7~13일 | 40 |
| 3~6일 | 25 |
| 3일 미만 | 10 |

#### CPC 점수

| 평균 CPC | 점수 | 의미 |
|----------|------|------|
| 5,000원 이상 | 90 | 광고 가치 높음 |
| 3,000~4,999 | 75 | 가치 있음 |
| 1,000~2,999 | 60 | 보통 |
| 500~999 | 45 | 낮음 |
| 100~499 | 30 | 매우 낮음 |
| 100 미만 | 15 | 거의 없음 |

### 등급 테이블

**점수가 높을수록 경쟁이 낮아 진입하기 좋다.** (D등급 = 추천, A등급 = 비추천)

| 등급 | 총점 범위 | 경쟁도 | 기회 평가 |
|------|-----------|--------|-----------|
| D- | 90 이상 | 경쟁 거의 없음 | 검색량 부족, 노출 효과 낮음 |
| D | 83~89 | 경쟁 매우 낮음 | 니치 키워드, 타겟 공략 |
| **D+** | **78~82** | **경쟁 낮음** | **진입 적합, 추천** |
| **C-** | **72~77** | **경쟁 다소 낮음** | **양호, 공략 가능** |
| C | 66~71 | 경쟁 보통 | 적정 경쟁, 도전 가능 |
| C+ | 60~65 | 경쟁 다소 높음 | 꾸준한 콘텐츠 필요 |
| B- | 52~59 | 경쟁 높음 | 차별화 필요 |
| B | 44~51 | 경쟁 높음 | 신중 검토 |
| B+ | 36~43 | 경쟁 매우 높음 | 비추천 |
| A- | 28~35 | 경쟁 치열 | 레드오션 |
| A | 20~27 | 경쟁 치열 | 과포화 |
| A+ | 20 미만 | 경쟁 극심 | 포화 시장 |

### 추천 분류

분석 결과는 자동으로 4단계 추천 그룹으로 분류된다.

| 그룹 | 등급 | 전략 |
|------|------|------|
| beginner (초보 추천) | D-, D, D+ | 진입 장벽 낮음, 빠른 노출 기대 |
| intermediate (중급) | C-, C, C+ | 적정 경쟁, 꾸준한 콘텐츠로 공략 |
| advanced (고급) | B-, B | 차별화 전략 필수, 경험자 추천 |
| avoid (비추천) | B+, A-, A, A+ | 레드오션, 진입 비추천 |

## 4. 트렌드 분석

### API: `/api/keywords/trends`

```
POST /api/keywords/trends
Content-Type: application/json
Authorization: Bearer {token}

{
  "keywords": ["마케팅 자동화", "AI 마케팅"],  // 1~5개
  "period": 12  // 1~24개월 (선택, 기본 12)
}
```

**데이터 소스:** 네이버 DataLab 검색어 트렌드 API (`openapi.naver.com/v1/datalab/search`)

**응답:**

```json
{
  "success": true,
  "data": {
    "results": [
      {
        "keyword": "마케팅 자동화",
        "data": [
          { "date": "2025-03", "ratio": 85 },
          { "date": "2025-04", "ratio": 92 }
        ]
      }
    ],
    "period": "2024-04-01 ~ 2025-04-01"
  }
}
```

**활용:** 트렌드 변화율(change)이 +5% 이상이면 `rising`, -5% 이하면 `declining`, 그 사이는 `stable`로 분류.

## 5. 기회 키워드 추천

### API: `/api/keywords/opportunities`

```
GET /api/keywords/opportunities?industry=음식업&limit=20
Authorization: Bearer {token}
```

| 파라미터 | 기본값 | 제약 | 설명 |
|----------|--------|------|------|
| industry | general | - | 업종 필터 |
| limit | 20 | 최대 50 | 반환 개수 |

**응답 필드:**

| 필드 | 설명 |
|------|------|
| keyword | 키워드 |
| opportunity_score | 기회 점수 (높을수록 좋음) |
| search_volume | 월간 검색량 |
| competition_level | 경쟁 등급 |
| trend_direction | rising / stable / declining |

**캐싱:** `keyword_opportunities` 테이블에 7일간 캐시. 만료 시 재계산.

## 6. 실전 활용 전략

### B2B 키워드 전략

| 단계 | 전략 | 예시 |
|------|------|------|
| 진입기 | D/D+ 등급 롱테일 키워드 집중 | "B2B SaaS 마케팅 자동화 도입 사례" |
| 성장기 | C등급 키워드 확장 | "마케팅 자동화 솔루션 비교" |
| 확장기 | B등급 도전 + D등급 니치 병행 | "마케팅 자동화" + 세부 롱테일 |

### B2C 키워드 전략

| 단계 | 전략 | 예시 |
|------|------|------|
| 진입기 | D+ 등급 감성/후기 키워드 | "강남 카페 숨은 맛집 후기" |
| 성장기 | C등급 시즌/트렌드 키워드 | "카페 추천 2025" |
| 확장기 | B등급 메인 키워드 도전 | "강남 카페" |

### 키워드 조합 공식

1. **메인 키워드** (C등급) 1개 선정
2. **롱테일 키워드** (D+~D등급) 3~5개 확보
3. **트렌드 상승** 키워드 1~2개 추가
4. 고정 키워드(fixedKeywords)로 등록하여 전 콘텐츠에 반영

## 7. 파이프라인 연결

STEP1 키워드 분석 결과는 이후 모든 단계에 `PipelineContext.keyword`로 전달된다.

```
STEP1 키워드 분석
  ↓ keyword, fixedKeywords
STEP2 글 유형 선택 (info/intro/service/product)
  ↓ topicType
STEP3 제목 5개 생성
```

**마이페이지 설정값 연동:**
- `industry` → 업종별 기회 키워드 필터
- `fixedKeywords` → 고정 키워드 (모든 콘텐츠에 반영)
- `blogCategory` → 네이버 블로그 33개 카테고리 중 선택
