# STEP2~5: 블로그 글 생성 가이드

> 키워드 분석(STEP1) 이후, 글 유형 선택 → 제목 생성 → 초안 작성 → 채널별 변환까지의 블로그 콘텐츠 생성 파이프라인.

---

## 1. 파이프라인 흐름

```
STEP2: 글 유형 선택 (topicType)
  ↓
STEP3: 제목 5개 생성 → 사용자 1개 선택
  ↓
STEP4: 핵심 초안 (백그라운드, 유저 비노출)
  ↓
STEP5: 채널별 변환 (블로그 → Threads → 인스타 → 페이스북)
```

---

## 2. STEP2: 글 유형 선택

파이프라인 시작 시 사용자가 4가지 글 유형 중 하나를 선택한다.

| topicType | 설명 | B2B 적합 | B2C 적합 |
|-----------|------|----------|----------|
| `info` | 정보형 (독자에게 유용한 정보 제공) | O | O |
| `intro` | 회사/브랜드 소개 | O | O |
| `service` | 서비스 소개 및 설명 | O | - |
| `product` | 상품/제품 소개 및 리뷰 | - | O |

---

## 3. STEP3: 제목 5개 생성

### API: `/api/generate/titles`

```
POST /api/generate/titles
Content-Type: application/json
Authorization: Bearer {token}

{
  "business_type": "B2C",
  "keyword": "카페 인테리어",
  "company_name": "카페모아",
  "service_name": "인테리어 컨설팅",
  "target_audience": "20~30대 카페 창업자",
  "tone": "친근한",
  "topic_type": "info",
  "custom_prompt": "트렌드 중심으로",   // 선택
  "prompt_mode": "combine"              // 선택
}
```

**요청 파라미터:**

| 필드 | 필수 | 타입 | 설명 |
|------|------|------|------|
| business_type | O | `B2B` \| `B2C` | 비즈니스 유형 |
| keyword | O | string | 핵심 키워드 |
| company_name | - | string | 회사/브랜드명 |
| service_name | - | string | 서비스/제품명 |
| target_audience | - | string | 타겟 고객 |
| tone | - | string | 글 톤 (친근한, 전문적인, 감성적인 등) |
| topic_type | - | `info` \| `intro` \| `service` \| `product` | 글 유형 |
| custom_prompt | - | string | 사용자 커스텀 프롬프트 |
| prompt_mode | - | `priority` \| `combine` \| `reference` | 프롬프트 적용 방식 |

**AI 모델:** `claude-sonnet-4-20250514` / max_tokens: 1024

**응답:**

```json
{
  "success": true,
  "data": {
    "titles": ["제목1", "제목2", "제목3", "제목4", "제목5"],
    "tokens": 432
  }
}
```

### B2B vs B2C 제목 차이

| 구분 | B2B | B2C |
|------|-----|-----|
| 톤 | 전문성과 신뢰감 | 클릭 유도, 감성적 공감 |
| 키워드 | 데이터, 사례, 효과, ROI | 후기, 추천, 꿀팁, 라이프스타일 |
| 예시 | "마케팅 자동화 도입 후 ROI 200% 달성 사례" | "직장인이 꼭 알아야 할 카페 인테리어 꿀팁 5가지" |

### 사용자 프롬프트 모드 (promptMode)

| 모드 | 동작 | 사용 시점 |
|------|------|-----------|
| `priority` | 사용자 프롬프트를 최우선 적용, 기본 가이드는 참고용 | 특정 방향성이 강할 때 |
| `combine` | 기본 가이드 + 사용자 프롬프트 합산 (기본값) | 일반적인 커스터마이징 |
| `reference` | 기본 가이드 우선, 사용자 프롬프트는 참고만 | 미세 조정 |

## 4. STEP4: 핵심 초안 생성

### API: `/api/generate/draft`

```
POST /api/generate/draft
Content-Type: application/json
Authorization: Bearer {token}

{
  "project_id": "uuid-..."
}
```

**특징:**
- 프로젝트에 저장된 `settings_snapshot`에서 모든 설정값을 로드
- 이미 초안이 있고 `step_status.s4 === 'completed'`이면 중복 생성 방지 (skipped: true 반환)
- **유저에게 직접 노출되지 않는 중간 산출물** — 채널별 변환의 원본 소스로 사용

**AI 모델:** `claude-sonnet-4-20250514` / max_tokens: 4096

### 초안 구조

| 구분 | B2B | B2C |
|------|-----|-----|
| 분량 | 2,000~3,000자 | 1,500~2,000자 |
| 구조 | 도입 / 본론(3~4개 소제목) / 결론 | 동일 |
| 키워드 | SEO 키워드 자연스럽게 3~5회 포함 | 동일 |
| B2B 특화 | 데이터와 사례 중심, 논리적 구조 | - |
| B2C 특화 | - | 스토리텔링과 공감 요소, 읽기 쉽게 |

### settings_snapshot에서 로드되는 값

| 필드 | PipelineContext 매핑 | 출처 |
|------|---------------------|------|
| company_name | companyName | 마이페이지 |
| service_name | serviceName | 마이페이지 |
| target_audience | targetAudience | 마이페이지 |
| target_gender | targetGender | 마이페이지 |
| writing_tone | tone | 마이페이지 |
| industry | industry | 마이페이지 |
| blog_category | blogCategory | 마이페이지 |
| fixed_keywords | fixedKeywords | 마이페이지 |

## 5. STEP5: 채널별 변환 (블로그)

### API: `/api/generate/pipeline`

```
POST /api/generate/pipeline
Content-Type: application/json
Authorization: Bearer {token}

{
  "project_id": "uuid-..."
}
```

**동작 방식:**
1. 프로젝트의 `settings_snapshot.selected_channels`에서 채널 목록 로드
2. **블로그가 포함되어 있으면 항상 첫 번째로 생성** (다른 채널이 블로그 결과를 참조)
3. SSE(Server-Sent Events) 스트리밍으로 채널별 진행 상황 실시간 전송
4. 각 채널 완료 시 `contents` 테이블에 개별 저장 + `projects.content_ids`에 매핑

### SSE 이벤트 타입

| type | 데이터 | 설명 |
|------|--------|------|
| `progress` | `{ channel, message }` | 생성 시작 알림 |
| `channel_done` | `{ channel, contentId, wordCount }` | 채널 완료 |
| `done` | `{ contentIds }` | 전체 완료 |
| `error` | `{ channel?, message }` | 오류 발생 |

### 블로그 채널 규격

| 항목 | B2B | B2C |
|------|-----|-----|
| 형식 | 네이버 블로그 (B2B 전문 콘텐츠) | 네이버 블로그 (B2C 감성 콘텐츠) |
| 분량 | 2,500~3,500자 | 1,800~2,500자 |
| 소제목 | H2/H3, 데이터 표, 인용 활용 | 읽기 쉬운 문단, 이모지 적절 활용 |
| 키워드 | 본문 내 5~8회 자연 포함 | 본문 내 3~5회 자연 포함 |
| 특화 | 사례/레퍼런스 1~2개 포함 | 후기/경험담 형식 추천 |
| CTA | 상담/문의 유도 | 구매/체험 유도 |

**AI 모델:** `claude-sonnet-4-20250514` / max_tokens: 4096 (블로그), 2048 (기타 채널)

## 6. 블로그 카테고리 (네이버 33개 토픽)

마이페이지에서 설정 가능한 블로그 카테고리. `blogCategory`로 프롬프트에 반영된다.

### 엔터테인먼트/예술

영화, 미술/디자인, 공연/전시, 음악, 드라마, 스타/연예인, 만화/애니, 방송

### 생활/노하우/쇼핑

일상/생각, 육아/결혼, 반려동물, 좋은글/이미지, 패션/미용, 인테리어/DIY, 요리/레시피, 상품리뷰, 원예/재배

### 취미/여가/여행

게임, 스포츠, 사진, 자동차, 취미, 국내여행, 세계여행, 맛집

### 지식/동향

IT/컴퓨터, 사회/정치, 건강/의학, 비즈니스/경제, 어학/외국어, 교육/학문

## 7. 업종별 맞춤 콘텐츠 생성

`getIndustryContext()` 함수가 프롬프트에 업종 정보를 주입한다.

```
[업종 맞춤 컨텍스트 - 카페/디저트]
- 핵심 가치: 분위기, 감성, 인스타그래머블, 맛+공간
- 주요 고객: 20~30대, 데이트, 소모임, 재택근무자
- 필수 요소: 매장 인테리어 사진, 시그니처 메뉴, 온라인 메뉴판, 위치+주차
- 차별화 포인트: "직접 로스팅한 원두", "매일 굽는 수제 디저트"
- 감정 트리거: 나만의 아지트, 특별한 하루
```

이 컨텍스트는 제목 생성(STEP3), 초안(STEP4), 채널별 변환(STEP5) 모두에 적용된다.

---

## 8. 고정 키워드 (fixedKeywords)

마이페이지에서 설정한 고정 키워드는 모든 콘텐츠에 자연스럽게 포함된다.

- 제목 생성 시: 고정 키워드 프롬프트에 명시
- 초안 작성 시: "본문에 자연스럽게 포함" 지시
- 채널 변환 시: 원본 초안에 이미 포함되어 자연스럽게 유지

## 9. 발행

### 클립보드 복사

```

```

## 10. 프로젝트 데이터 흐름

```
projects 테이블:
  ├─ keyword_text      ← STEP1에서 설정
  ├─ topic_type        ← STEP2에서 선택
  ├─ selected_title    ← STEP3에서 사용자 선택
  ├─ draft_content     ← STEP4에서 AI 생성 (비노출)
  ├─ content_ids       ← STEP5에서 채널별 contents.id 매핑
  ├─ settings_snapshot ← 프로젝트 생성 시 마이페이지 설정 스냅샷
  ├─ current_step      ← 현재 진행 단계 (1~7)
  └─ step_status       ← { s4: 'completed', s5: 'completed', ... }
```
