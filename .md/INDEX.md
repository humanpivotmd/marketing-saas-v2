### 문서 운영 원칙

1. **코드에서 읽을 수 있는 것은 문서로 만들지 않는다** (타입, 컴포넌트 props, 환경변수, git log)
2. **같은 정보는 1곳에만** — 중복 발견 시 한쪽 삭제, 다른 쪽에서 참조
3. **코드 변경 시 관련 .md 확인** — 영향 범위 파악 후 함께 수정
4. **작업이력.md에 기록** — 매 세션 종료 시
