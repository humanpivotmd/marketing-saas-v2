# v3 보안 체크리스트

## 1. 인증 (requireAuth)

- 모든 v3 API에 `requireAuth(req)` 적용
- Bearer 토큰 기반 JWT 인증
- JWT_SECRET: 32자 이상 필수 (Zod 검증)
- 인증 실패 시 401 응답

적용 API:
- /api/projects (GET, POST)
- /api/projects/:id (GET, PATCH, DELETE)
- /api/generate/titles (POST)
- /api/generate/draft (POST)
- /api/generate/pipeline (POST)
- /api/generate/image-script (POST)
- /api/generate/video-script (POST)
- /api/mypage/business-profile (GET, PUT)
- /api/industries (GET)
- /api/calendar (GET)

## 2. 소유권 검증 (user_id)

모든 데이터 조회/수정/삭제에 `user_id = authUser.userId` 조건 적용:

| API | 소유권 검증 방식 |
|-----|----------------|
| GET /api/projects | `.eq('user_id', authUser.userId)` |
| GET /api/projects/:id | `.eq('user_id', authUser.userId)` |
| PATCH /api/projects/:id | `.eq('user_id', authUser.userId)` |
| DELETE /api/projects/:id | `.eq('user_id', authUser.userId)` |
| POST /api/generate/draft | 프로젝트 조회 시 `.eq('user_id', authUser.userId)` |
| POST /api/generate/pipeline | 프로젝트 조회 시 `.eq('user_id', authUser.userId)`, 콘텐츠 insert 시 user_id 설정 |
| POST /api/generate/image-script | 프로젝트 조회 + 콘텐츠 조회 + 기존 삭제 모두 user_id 검증 |
| POST /api/generate/video-script | 프로젝트 조회 + 기존 삭제 모두 user_id 검증 |
| GET/PUT /api/mypage/business-profile | 자기 자신 데이터만 |
| GET /api/calendar | `.eq('user_id', user.userId)` |

추가로 DB 레벨에서 RLS 정책으로 이중 보호:
- projects: `user_id = auth.uid()`
- image_scripts: `user_id = auth.uid()`
- video_scripts: `user_id = auth.uid()`
- user_prompts: `user_id = auth.uid()`

## 3. Rate Limit (generateRateLimit)

파일: `src/lib/rate-limit.ts`

- 구현: 인메모리 Map 기반 (IP:path 키)
- `generateRateLimit`: 분당 10회 제한
- 적용 API: /api/generate/titles, /api/generate/draft, /api/generate/pipeline, /api/generate/image-script, /api/generate/video-script
- 초과 시 429 응답 + `Retry-After` 헤더
- 5분마다 만료된 엔트리 정리 (setInterval)
- IP 추출: x-forwarded-for -> x-real-ip -> 127.0.0.1

응답 헤더:
```
HTTP 429
Content-Type: application/json
Retry-After: <seconds>
X-RateLimit-Limit: 0
X-RateLimit-Remaining: 0
```

## 4. 입력 검증

### sanitizeInput (src/lib/sanitize.ts)
HTML 태그/이벤트 핸들러 제거:
- `<script>` 태그 제거
- `on*=""` 이벤트 핸들러 제거
- `<iframe>`, `<object>`, `<embed>` 제거
- `javascript:` 프로토콜 제거

적용: /api/generate/titles (keyword, company_name, service_name, target_audience)

### sanitizePromptInput (src/lib/sanitize.ts)
프롬프트 인젝션 방지:
- 차단 패턴: `ignore.*previous.*instructions`, `ignore.*above`, `disregard.*instructions`, `forget.*everything`, `system\s*prompt`
- 매칭 시 `[blocked]`로 치환
- 최대 2000자 제한 (slice)

적용: /api/generate/titles (custom_prompt)

### Zod 검증 (src/lib/env.ts)
환경변수 유효성 검증:
- NEXT_PUBLIC_SUPABASE_URL: URL 형식 필수
- JWT_SECRET: 32자 이상 필수
- production 환경에서 검증 실패 시 throw

### 서버측 유효성 검사
- /api/mypage/business-profile PUT: business_type는 B2B/B2C만 허용, target_gender는 male/female/all만 허용
- /api/generate/titles POST: keyword 필수 검증
- /api/generate/draft POST: project_id 필수 검증
- /api/generate/image-script POST: project_id + channel 필수 검증
- /api/generate/video-script POST: project_id 필수 검증
- /api/projects/:id PATCH: 허용 필드 화이트리스트 방식 (13개 필드: current_step, step_status, topic_type, selected_title, title_candidates, custom_prompt, prompt_mode, draft_content, content_ids, status, confirmed_at, business_type, core_message)

## 5. 에러 처리

- 모든 API: try/catch + `handleApiError(err)` 패턴
- production에서는 상세 에러 메시지 숨김 (src/lib/errors.ts)
- AI API 실패 시 502 응답 + 사용자 친화적 메시지
- SSE 스트림에서 채널별 에러 시 해당 채널만 건너뛰고 계속 진행

## 6. 마이페이지 필수 설정 강제

`SetupRequired` 컴포넌트로 /create/draft-info 진입 시 강제:
- 확인 항목: business_type, selected_channels(1개 이상), company_name
- sessionStorage 캐시: `business_setup` 키로 done/needed 저장
- 미설정 시 /settings#business로 유도
