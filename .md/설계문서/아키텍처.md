# 아키텍처

> MarketingFlow v2 -- Next.js 16 (App Router) + Supabase + Tailwind CSS

---

## 1. 기술 스택

| 계층 | 기술 |
|------|------|
| 프론트엔드 | Next.js 16 App Router, React, Tailwind CSS |
| 백엔드 | Next.js API Route Handlers (68개) |
| DB | Supabase (PostgreSQL), RLS 전체 적용 |
| AI | Claude API (claude-sonnet-4-20250514), Gemini 폴백 |
| 이메일 | Resend API |
| 키워드 | 네이버 검색광고 API, 네이버 DataLab API |
| 소셜 발행 | Meta Graph API (Instagram, Threads) |
| 결제 | 토스페이먼츠 (준비됨, 미연동) |
| 인증 | JWT (jsonwebtoken) + bcrypt + HttpOnly 쿠키 |

## 2. 핵심 흐름도

```
[사용자] → 로그인(JWT) → 마이페이지 설정(비즈니스 프로필)
                              ↓
                     키워드 분석 (네이버 API)
                              ↓
                     7단계 파이프라인 (Claude API)
                              ↓
                     콘텐츠 편집/확인 → 발행 (Meta API / 클립보드)
```

## 4. 7단계 파이프라인

```
STEP 1: 키워드 선택/분석    → keywords 테이블 + 네이버 API
STEP 2: 설정 확인           → users 비즈니스 프로필 → settings_snapshot
STEP 3: 제목 생성 (5개)     → Claude API
STEP 4: 핵심 초안           → Claude API (유저 비노출, 소스 콘텐츠)
STEP 5: 채널별 변환 (SSE)   → Claude API → blog/threads/instagram/facebook
STEP 6: 이미지 프롬프트     → Claude API → Midjourney/DALL-E 프롬프트
STEP 7: 영상 스토리보드     → Claude API → 장면별 스크립트
```

- 프로젝트 단위 관리: `projects` 테이블의 `current_step`, `step_status`(JSONB)로 진행 상태 추적
- STEP 5에서 blog를 항상 먼저 생성하고, 그 결과를 다른 채널의 컨텍스트로 전달
- 채널별 분량 규격은 B2B/B2C와 채널 종류에 따라 다름 (코드 참조: `src/lib/prompts/pipeline.ts` CHANNEL_SPECS)

## 5. 인증 흐름

```
[클라이언트] ── Bearer Token ──→ [API Route] ── requireAuth() ──→ [비즈니스 로직]
                                                   │
                                        verifyToken(JWT) → { id, userId, role }
```

- JWT 24시간 만료, Authorization 헤더 + HttpOnly 쿠키 이중 전달
- 비밀번호: bcrypt 12 rounds
- 역할: user, member, team_admin, admin, super_admin
- 관리자 API: `requireAuth` + `requireAdmin` 패턴
- 상세 보안 사항 → `보안.md` 참조

## 6. 외부 API 연동

### Claude API (Anthropic)
- AI 콘텐츠 생성 전 과정 (제목, 초안, 채널변환, 이미지/영상 스크립트)
- 프롬프트: admin_prompts 테이블 (A/B 테스트) + user_prompts (커스텀) + INDUSTRY_CONTEXT (30개 업종)
- 코드 참조: `src/app/api/generate/` (9개 엔드포인트), `src/lib/prompts/pipeline.ts`

### 네이버 API (검색광고 + DataLab)
- 키워드 검색량/CPC/경쟁도 조회 (HMAC-SHA256 서명 인증)
- 90일 트렌드 분석, 블로그 발행량/노출일 조회 → 기회 등급(A+~D-) 산출
- 코드 참조: `src/lib/naver.ts`, `src/lib/grade.ts`

### Meta API (Instagram / Threads)
- 소셜 게시물 자동 발행, 채널별 access_token AES-256-GCM 암호화 저장
- 코드 참조: `src/app/api/publish/instagram/`, `src/app/api/publish/threads/`

### Resend API (이메일)
- 이메일 인증(24h), 비밀번호 재설정(1h), 관리자 브로드캐스트
- 코드 참조: `src/lib/email.ts`

## 7. 주요 컴포넌트

### FlowGuard (`src/components/FlowGuard.tsx`)
프로젝트 단계 진행 가드. 이전 단계가 완료되지 않으면 `/create/*` 페이지 접근을 차단한다.
- 상태: `loading` | `ok` | `no_project` | `step_not_ready`

### SetupRequired (`src/components/SetupRequired.tsx`)
마이페이지 비즈니스 설정 미완료 시 키워드 분석/콘텐츠 생성 페이지 진입을 차단하고 설정 페이지로 안내한다.
- 필수 항목: `business_type`, `industry_id`, `selected_channels`
- 캐싱: `sessionStorage.business_setup`

> UI 컴포넌트(Button, Card, Modal 등)는 코드 참조: `src/components/ui/`

## 8. 설정 체인

마이페이지에서 한 번 설정하면 모든 콘텐츠 생성에 자동 반영되는 데이터 흐름:

```
[마이페이지 비즈니스 프로필]   ← users 테이블 v3 컬럼
          ↓
[프로젝트 생성 시 스냅샷]      ← projects.settings_snapshot (JSONB)
          ↓
[PipelineContext로 변환]       ← buildTitlePrompt(), buildDraftPrompt(), buildChannelPrompt()
          ↓
[AI 프롬프트에 컨텍스트 주입]  ← 업종, B2B/B2C, 톤, 타겟, 키워드 등
```

PipelineContext 주요 필드: `businessType`, `keyword`, `companyName`, `serviceName`, `targetAudience`, `tone`, `industry`, `topicType`, `promptMode`

코드 참조: `src/lib/prompts/pipeline.ts` (`PipelineContext` 인터페이스)
