# Agent Development Kit

## 개요
Agent Development Kit는 AI 에이전트를 체계적으로 개발하고 운영하기 위한 아키텍처 프레임워크입니다.

## 핵심 구성 요소

### 1. CLAUDE.md - 헌법/규칙 정의
- **역할**: 프로젝트의 근본 규칙과 운영 원칙 정의
- **내용**: 금지 파일, 작업 플로우, 품질 기준, 보안 규칙
- **위치**: 프로젝트 루트
- **업데이트**: 프로젝트 요구사항 변경 시

### 2. SKILLS - 실행 가능한 작업 단위
- **역할**: 실제 작업을 수행하는 모듈
- **구성**: /spec, /plan, /build, /test, /review, /ship
- **특징**: 각 단계별 전문성 캡슐화
- **호출**: 명령어 기반 자동 실행

### 3. HOOKS - 품질 게이트
- **역할**: 실행 전/후 검증 및 제어 + **자동화된 영향 분석**
- **기능**:
  - **사전 검증**: 권한, 리소스, CLAUDE.md 준수 확인
  - **영향 분석**: 파일 의존성, UI/프로세스/함수/DB 영향 자동 스캔
  - **충돌 감지**: 코드 중복, 네이밍 충돌, 인터페이스 불일치
  - **품질 게이트**: 린팅, 타입 체크, 테스트 커버리지
  - **실행 제어**: 승인되지 않은 변경 차단
- **타이밍**: 각 SKILL 실행 전/후 + 실시간 모니터링

### 4. SUBAGENTS - 분산 처리
- **역할**: 복잡한 작업을 하위 에이전트에게 위임 + **전문화된 영향 분석**
- **장점**: 병렬 처리, 전문성 분배, 복합 분석
- **전용 SUBAGENTS**:
  - **Impact Analyzer**: UI/UX, 프로세스, 함수, DB 영향 분석
  - **Dependency Scanner**: 파일 의존성, 호출 체인 자동 추적
  - **Risk Assessor**: 복합 리스크 평가 및 완화 전략
  - **Code Reviewer**: 품질 검토 및 개선 제안
  - **Test Generator**: 영향받은 영역의 테스트 자동 생성

### 5. PLUGINS - 외부 기능 연동
- **역할**: 실제 시스템 연동 (파일 수정, API 호출, 빌드 실행)
- **형식**: npm 패키지 또는 모듈
- **배포**: 독립적 업데이트 가능

## 개발 원칙

### 1. 계층 분리
- 각 계층은 독립적 책임
- 인터페이스 표준화
- 교체 가능성 보장

### 2. 품질 우선
- HOOKS를 통한 검증
- SKILLS의 전문성
- CLAUDE.md의 규칙 준수

### 3. 반복 가능성
- 동일 입력 → 동일 출력
- 상태 관리 표준화
- 에러 처리 일관성

### 4. 확장성
- PLUGINS를 통한 기능 추가
- SUBAGENTS를 통한 규모 확장
- SKILLS 모듈화
