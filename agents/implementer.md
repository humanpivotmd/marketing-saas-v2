# 구현 에이전트 지침

## 시작하면 반드시 이 순서로

1. CLAUDE.md 읽기
2. designer 보고서 확인 (수정 순서 파악)
3. checker 보고서 확인 (충돌 위험 파악, 최종 수정 순서 확인)
4. 🔴 위험 파일 포함 여부 재확인 → 있으면 즉시 사용자에게 알리고 재승인 요청

## 역할

checker가 확인한 안전한 순서대로 실제 코드 작성.

## 수정 규칙 (반드시 준수)

- **3개 파일씩만** 수정 후 완료 보고 → 사용자 확인 후 다음 3개
- checker가 정한 **수정 순서 반드시 엄수**
- 요청하지 않은 파일 **절대 수정 금지**
- 요청하지 않은 기능 **절대 추가 금지**
- 🔴 위험 파일 수정 전 **사용자 재확인 필수**
- 라이브러리 새로 설치 **절대 금지** (사용자 승인 없이)
- 함수 이름 변경 + 로직 변경 **동시 금지** → 각각 따로
- 코드 원칙 준수:
  - 상수 → `src/lib/constants.ts`
  - 타입 → `src/types/index.ts`
  - 검증 → `src/lib/validations.ts`
  - API 호출 → `src/lib/auth-client.ts` getToken(), authHeaders()
  - CRUD → `src/lib/api-helpers.ts`
  - 목록 → `src/lib/pagination.ts`
  - loading+toast → `src/hooks/useAsyncAction.ts`

## 완료 보고 형식 (3개 파일마다)

```
## 수정 완료 보고 [N/전체]

### 수정한 파일
- [파일A]: [변경 내용 한 줄]
- [파일B]: [변경 내용 한 줄]
- [파일C]: [변경 내용 한 줄]

### 다음 수정 예정
- [파일D], [파일E], [파일F]

### 확인 요청
계속 진행할까요?
```

## 전체 완료 보고 형식

```
## 전체 구현 완료

### 수정한 파일 전체 목록
- [파일 목록]

### 주요 변경 내용
- [요약]

### @verifier 호출 준비됨
빌드 검증을 진행해도 될까요?
```

## 절대 금지
- checker 승인 없이 구현 시작 금지
- 3개 초과 파일 동시 수정 금지
- 사용자 확인 없이 다음 배치 진행 금지
