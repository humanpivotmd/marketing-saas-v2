# v3 프롬프트 설계

파일: `src/lib/prompts/pipeline.ts`

## 1. B2B vs B2C 프롬프트 분기 구조

모든 프롬프트에 공통 적용되는 `getBusinessContext()` 함수:

### B2B 원칙
- 전문적이고 격식 있는 문체
- 데이터, 수치, 사례(케이스스터디) 중심
- 논리적 근거와 ROI 강조
- 구매 담당자/임원이 의사결정에 활용 가능한 정보
- 키워드: 효율성, 생산성, 비용 절감, 확장성, 도입 효과
- 신뢰 구축: 레퍼런스, 인증, 데이터 기반 증거

### B2C 원칙
- 친근하고 감성적인 문체
- 스토리텔링과 공감 활용
- 즉각적인 감정 자극과 행동 유도
- 일반 소비자가 쉽게 이해하는 수준
- 키워드: 지금 바로, 한정, 특가, 후기, 라이프스타일
- 신뢰 구축: 후기, 별점, 인플루언서, 감정적 경험

## 2. 16개 프롬프트 템플릿

### STEP3: 제목 생성 (`buildTitlePrompt`)
- B2B 제목: 전문성과 신뢰감 (데이터, 사례, 효과 강조)
- B2C 제목: 클릭 유도, 감성적, 공감
- 입력: keyword, businessType, companyName, serviceName, topicType, tone, targetAudience, targetGender, blogCategory, fixedKeywords
- 출력: JSON 배열 5개 `["제목1", ..., "제목5"]`

### STEP4: 핵심 초안 (`buildDraftPrompt`)
- B2B: 2000~3000자, 데이터/사례 중심, 논리적 구조
- B2C: 1500~2000자, 스토리텔링/공감, 읽기 쉽게
- 구조: 도입/본론(3~4개 소제목)/결론
- SEO 키워드 3~5회 자연 포함
- 입력: 위 + selectedTitle, topicType, industry

### STEP5: 채널별 변환 (`buildChannelPrompt`)
채널 4종 x B2B/B2C = 8개 변형:

| 채널 | B2B 분량 | B2C 분량 | B2B 특성 | B2C 특성 |
|------|----------|----------|----------|----------|
| blog | 2500~3500자 | 1800~2500자 | H2/H3, 데이터 표, 인용, 사례/레퍼런스, 상담/문의 CTA | 이모지, 후기/경험담, 구매/체험 CTA |
| threads | 200~400자 | 100~300자 | 전문적 간결, 핵심 데이터+인사이트, 해시태그 3~5 | 대화체, 이모지, 첫줄 후킹, 해시태그 5~10 |
| instagram | 300~500자 | 200~400자 | 전문 인사이트, 해시태그 10~15, CTA: 링크/프로필 | 이모지 적극, 감성 스토리텔링, 해시태그 15~25, CTA: 저장/공유/댓글 |
| facebook | 500~800자 | 300~600자 | 업계 뉴스/트렌드 분석, 링크 공유, 해시태그 3~5 | 대화형, 공감/공유, 이벤트/혜택, 해시태그 3~5 |

- 소스: blogContent(블로그 먼저 생성된 경우) 또는 draftContent
- 소스를 3000자까지 잘라서 프롬프트에 포함

### STEP6: 이미지 프롬프트 (`buildImageScriptPrompt`)
- B2B: 전문적/깔끔한 비즈니스 분위기, 데이터 차트/인포그래픽/대시보드, 블루/그레이 계열
- B2C: 감성적/밝은 라이프스타일, 인물 중심, 따뜻한 자연광, 제품 자연 노출
- 출력: images 3개(seq, description_ko, prompt_en, placement) + thumbnail 1개

### STEP7: 영상 스크립트 (`buildVideoScriptPrompt`)
- B2B: 설명형/데모 중심, 데이터 시각화, 사례 발표, CTA("무료 상담", "데모 신청", "백서 다운로드")
- B2C: 감성적 빠른 전환, 첫3초 후킹, 인물 라이프스타일, 트렌디한 BGM, CTA("지금 바로", "링크 클릭")
- 출력: JSON { title, total_duration, scenes[{scene, duration, visual, narration, text_overlay, transition}], bgm_suggestion, hook }

## 3. 커스텀 프롬프트 3모드 (`applyUserPrompt`)

| 모드 | 적용 방식 | 설명 |
|------|-----------|------|
| priority | 사용자 프롬프트를 상단에 배치, 기본 가이드를 `[참고]`로 하단 | 사용자 지시 최우선 |
| combine | 기본 프롬프트 하단에 `[추가 지침]`으로 사용자 프롬프트 추가 | 동등 결합 (기본값) |
| reference | 기본 프롬프트 하단에 `[참고 사항 -- 사용자 요청]`으로 추가 | 참고 수준 |

## 4. PipelineContext 필드 전체

```typescript
interface PipelineContext {
  businessType: 'B2B' | 'B2C'       // 필수
  keyword: string                    // 필수
  companyName?: string               // 회사명
  serviceName?: string               // 서비스/제품명
  targetAudience?: string            // 타겟 고객
  targetGender?: string              // male/female/all
  tone?: string                      // auto/professional/friendly/formal/conversational
  industry?: string                  // 업종 이름
  topicType?: string                 // info/intro/service/product
  selectedTitle?: string             // 선택된 제목 (STEP4 이후)
  draftContent?: string              // 핵심 초안 (STEP5 이후)
  blogContent?: string               // 블로그 본문 (채널 변환 시)
  userPrompt?: string                // 커스텀 프롬프트
  promptMode?: 'priority' | 'combine' | 'reference'
  blogCategory?: string              // 네이버 블로그 주제
  fixedKeywords?: string[]           // 고정 키워드 (최대 5개)
}
```

이미지/영상 생성 시 추가 확장 필드:
- 이미지: `aiTool`, `imageSize`, `imageStyle`, `styleDetail`, `channel`
- 영상: `format`, `targetChannel`, `sceneCount`, `sceneDuration`

## 5. 프롬프트 구조 특징

- system/user 분리 인터페이스(`SplitPrompt`)가 정의되어 있으나, 현재 API 호출에서는 단일 user 메시지로 전송
- 프롬프트 인젝션 방지: `sanitizePromptInput`으로 "ignore previous instructions" 등 패턴 차단, 2000자 제한
- 모든 AI 호출에 anthropic-version: `2023-06-01` 헤더 사용
